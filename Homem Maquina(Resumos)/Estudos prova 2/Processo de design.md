- Entrada -> Processo -> Output
	1. Análise: Quebrar um problema em menores para entender ele melhor (Chuncking)
	2. Síntese: Reagrupar os problemas menores para criar uma solução
- Alguns modelos acreditam que esse processo é linear (definir o problema -> resolvelo) e outros que entender e resolver um problema acontecem ao mesmo tempo
#### Evolução do Design
- Design clássico: Uma única maneira e "perfeita" para resolver, começou com os militares
- Design Thinkings: Executar(fazer) ultrapassou a inovação (fazer), foca na gestão de qualidade e na experiência de usuário, começou no mundo de negócios 
- Design Computacional: Projetar em larga escala e em tempo real, começou com software

#### O framework do duplo diamante
- Explorar amplamente um problema antes de gerar uma solução
- As 4 fases do diamante:
	- Descoberta: Criar opções (Pensamento divergente), começar com um problema (explorar ele), onde se tem ideias iniciais, inspirações e as necessidades dos usuários. Pesquisa, personas, cenários e mapas conceituais
	- Definição: Fazer escolhas(Pensamento convergente), interpretar os dados da descoberta e alinhar com o objetivo do projeto, define um PROBLEMA ESPECÍFICO
	- Desenvolvimento: Criar opções (divergente), com o problema definido é aberto um leque de possíveis soluções. Analise de tarefa, wireframe (esboço), mockups(representações visuais de alta/média fidelidade)
	- Entrega: Fazer escolhas (convergente), o desfecho do processo a melhor solução é escolhida, desenvolvida, finalizada, implementada e entregue. Relatórios e especificações

#### O que são testes de usabilidade?
- Usabilidade padrão
- Layout e consistencia 
- acessibilidade
- feedback do usuário
- desempenho de UI
#### O mapa dos métodos de teste
- Eixo Y: 
	- Comportamental (o que usuários fazem)
	- Atitudinal (o que os usuários falam)
- Eixo X:
	- Qualitativo(Por que?)
	- Quantitativo (Quantos?)
#### Os 5 tipos de teste de UI
- Testes de usabilidade padrão: Identificar problemas de navegação
- Testes de layout e consistência: Avaliar a organização dos elementos
- Testes de acessibilidade: Inclusão
- Testes com feedback do usuário: Coletar opiniões e sugestões dos usuários
- Testes de desempenho de UI: Testa em diferentes dispositivos e redes (velocidade, responsividade e estabilidade

#### 80/20 princípio de Pareto:
- Análise de tarefas: fluxo de trabalho + objetos de domínio. Sequência de atividades e as coisas que o usuário manipula na tela
- Dividir e conquistar 
- Analise vs Projeto: 
	- Analise: Identificar, garantir que o fluxo bata com as metas do usuário
	- Projeto: Solução técnica
